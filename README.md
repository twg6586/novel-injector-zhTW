# Novel Injector

SillyTavern 小说上下文注入插件。它可以把长篇小说整理成可复用的剧情、角色、阶段和向量索引，并在对话时自动注入相关原著信息，让 AI 续写时更贴合原作时间线、人物关系和剧情位置。

## 适合做什么

- 把 `.txt` 小说导入 SillyTavern，并按段落清洗压缩。
- 从小说中提取剧情节点、角色人设、世界设定和文风指南。
- 按剧情阶段管理上下文，选择哪些阶段参与注入。
- 对指定阶段建立 Embedding 向量索引，在聊天时召回最相关片段。
- 在未向量化阶段直接注入剧情节点或压缩原文。
- 保存多本小说快照，支持导入、导出和跨设备迁移。
- 使用“穿书模式”追踪当前剧情节点，并让 AI 围绕用户推进故事。
- 在穿书模式中按需开启“归纳”和“沉浸”，辅助召回并限制 AI 只写用户在剧情内可知的信息。

## 安装

### 方式一：通过 SillyTavern 扩展安装器

如果你的 SillyTavern 版本支持从 GitHub 安装第三方扩展，可以在扩展管理中安装：

```text
https://github.com/ZZZa-o/novel-injector
```

安装后刷新页面，点击顶部的书本图标打开插件。

### 方式二：手动安装

1. 下载本仓库。
2. 将整个 `novel-injector` 文件夹放入 SillyTavern 的第三方扩展目录：

```text
SillyTavern/public/scripts/extensions/third-party/novel-injector
```

3. 重启或刷新 SillyTavern。
4. 点击顶部书本图标打开插件面板。

## 快速开始

1. 在“处理”页上传小说 `.txt` 文件。
2. 填写清洗 API 配置，点击“开始全自动清洗”。
3. 清洗完成后，到“剧情”页检查剧情节点。
4. 到“阶段”页手动划分阶段，或按转折点自动划分。
5. 在“处理”页按需选择阶段并开始向量化。
6. 在“阶段”页开启需要注入的阶段。
7. 在 SillyTavern 正常聊天，插件会在发送消息时自动注入相关上下文。
8. 在“设置”页的小说库中保存快照，方便下次继续使用。
9. 如使用“穿书模式”，可在设置页开启“归纳”增强语义召回，或开启“沉浸”避免提前点破用户尚未知晓的人物姓名、身份和秘密。

## API 配置

插件内有两类 API 配置。

| 配置 | 用途 | 是否必需 |
|---|---|---|
| 清洗 API | 清洗压缩、提取剧情节点、角色人设、世界设定、文风、偏差分析等 | 必需 |
| Embedding API | 建立向量索引、聊天时语义召回相关片段 | 可选 |

清洗 API 兼容 OpenAI 风格的 Chat Completions 接口。Embedding API 兼容 OpenAI 风格的 Embeddings 接口。

不使用向量化也可以正常使用插件。未向量化阶段会按设置直接注入剧情节点或压缩原文。

## 核心概念

### 清洗压缩

上传小说后，插件会先按设定大小分段，再调用清洗 API 压缩正文并提取结构化信息。清洗结果会用于后续剧情节点、角色人设、阶段划分和注入。

### 剧情节点

剧情节点是从小说中提取出的事件摘要，包含时间、地点、人物和事件信息。它们适合低 token 成本注入，也适合用来划分阶段。

### 阶段

阶段是你希望插件管理的剧情区间。每个阶段可以单独开启或关闭注入，也可以单独向量化。聊天时，插件只会使用已开启阶段的内容。

### 向量化

向量化会把指定阶段的压缩文本切成小块并写入浏览器 IndexedDB。聊天时，插件根据最近消息生成查询向量，召回最相关的原著片段。

向量化适合文本量较大、希望召回更精确的阶段。对于短阶段或刚开始测试的用户，可以先不向量化，直接使用剧情节点注入。

### 角色人设

插件会从清洗结果中提取角色的原始人设，也支持根据当前对话更新 AI 实时人设。你可以在“角色”页选择哪些角色参与注入。

### 世界设定与文风

世界设定用于约束背景规则、社会结构、特殊机制等内容。文风指南用于帮助 AI 模仿原著的叙事节奏、描写偏好和语言习惯。

### 穿书模式

穿书模式会围绕当前剧情节点推进故事。它可以显示节点进度，让用户手动勾选已完成节点，并在聊天时注入当前叙事阶段相关提示。

穿书模式设置中提供两个辅助开关：

| 开关 | 作用 |
|---|---|
| 归纳 | 每轮回复自动附加对用户不可见的场景标签，供后续语义召回使用。开启后会把 `ni_query` 追加到“召回时提取的消息内容”。 |
| 沉浸 | 追加信息可见性边界，让 AI 区分后台原著资料、角色已知信息和 `<user>` 在剧情内已经知道的信息。开启后，正文会优先呈现 `<user>` 已感知、被告知或可合理推断的内容，未揭示的人名、身份、关系和秘密需要通过剧情内线索自然出现。 |

穿书模式的推进、持续、归纳、沉浸和后文推演提示词均可在“提示词”面板中编辑，并可单独恢复默认。

## 注入方式

插件发送消息时会按设置注入多类内容：

| 内容 | 说明 |
|---|---|
| 阶段剧情 | 未向量化阶段的剧情节点或压缩原文 |
| 向量召回 | 已向量化阶段中最相关的原著片段 |
| 角色人设 | 已启用角色的身份、性格、关系和当前状态 |
| 世界设定 | 世界边界、机制、规则等背景信息 |
| 文风指南 | 原著写作风格的执行规则 |
| 全局提示词 | 每次插件调用 AI 时前置的通用提示词 |
| 穿书提示 | 穿书模式下的当前节点、推进规则、叙事约束，以及可选的归纳/沉浸边界 |

注入位置、深度和角色可以在插件设置中调整。

## 小说库与导入导出

插件支持把当前工作区保存为小说快照。快照包含清洗文本、剧情节点、角色、阶段、世界设定、文风和向量状态等信息。

| 操作 | 说明 |
|---|---|
| 保存快照 | 把当前小说工作区保存到小说库 |
| 加载快照 | 切换到某个已保存小说 |
| 更新快照 | 用当前工作区覆盖当前快照 |
| 重命名 | 修改小说库中显示的快照名 |
| 删除 | 删除快照和对应向量缓存 |
| 导出 ZIP | 导出设置、运行时数据和向量数据 |
| 导入 ZIP / JSON | 导入旧版或新版备份，作为新快照加入小说库 |

导出的 ZIP 不包含 API Key。

## 数据存储与隐私

插件会把不同类型的数据存到不同位置：

| 数据 | 存储位置 |
|---|---|
| 轻量设置、小说库索引、阶段状态、世界设定索引等 | SillyTavern `extension_settings` |
| 剧情节点、角色人设、分段 meta、清洗状态、文风正文等 core 数据 | SillyTavern `user/files` 下的 `*_core.json` |
| 清洗后的压缩正文 | SillyTavern `user/files` 下的 `*_chunks.json` |
| 向量索引 | 浏览器 IndexedDB，数据库名为 `NovelInjectorVectors` |

说明：

- API Key 保存在 SillyTavern 设置中，不会写入导出 ZIP。
- 小说原文和清洗后的文本属于用户数据，请自行注意备份和隐私。
- 向量数据按小说 `novelKey` 隔离，不同小说之间不会混用。
- 加载快照时，插件会校准向量完成状态，避免显示“已向量”但本机没有实际向量数据。

## 文件命名

服务端重数据文件名使用：

```text
ni_<快照名连续拼音>_<随机key>_core.json
ni_<快照名连续拼音>_<随机key>_chunks.json
```

例如：

```text
斗破苍穹 → ni_doupocangqiong_171xxxxxxx_abcde_core.json
斗破苍穹 → ni_doupocangqiong_171xxxxxxx_abcde_chunks.json
```

这样做是为了兼容 SillyTavern 文件接口对文件名字符的限制，同时让用户在文件夹中也能大致看懂是哪本小说。

加载快照时，插件会优先读取 `core.json`。压缩正文体积较大，只有在压缩原文注入、向量化、导出备份等场景才会读取 `chunks.json`。

## 常见问题

### 插件图标没有出现怎么办？

确认插件目录是否放在：

```text
SillyTavern/public/scripts/extensions/third-party/novel-injector
```

并确认该目录下存在 `manifest.json` 和 `index.js`。然后刷新或重启 SillyTavern。

### 清洗失败怎么办？

优先检查清洗 API 的端点、Key、模型名和限速设置。单段失败后，可以点击该段的“生成此段”单独重试，也可以使用“重试失败分段”批量重跑。

### 向量化失败怎么办？

检查 Embedding API 的端点、Key 和模型名。若遇到限速，降低每分钟请求数后重试。可以使用向量诊断查看 IndexedDB 中实际保存了多少向量块。

### 已向量化后修改阶段，需要重新向量化吗？

需要。阶段划分变化后，受影响阶段的向量状态会重置为未向量。重新选择这些阶段并向量化即可。

### 剧情节点模式和压缩原文模式有什么区别？

剧情节点模式更省 token，结构更清晰；压缩原文模式信息更完整，但 token 消耗更高。可以在阶段页切换未向量阶段的注入模式。

切换到压缩原文模式后，插件会读取对应的 `chunks.json`，再刷新阶段 token 估算。

### 导入备份会覆盖当前小说吗？

不会。导入内容会作为新快照加入小说库。你需要手动加载该快照，才会切换当前工作区。

### 清除全部数据会删除哪些内容？

会删除当前小说的向量缓存、服务端 `core.json` / `chunks.json`，并移除当前小说在小说库中的条目。删除前会二次确认。

### 为什么文风采样刷新后不可用？

文风采样依赖当前会话中的原始上传分段。刷新页面后，原始上传分段可能不在内存中。可以重新上传文件，或使用范文模式手动粘贴样本文本。

## 文件结构

```text
novel-injector/
├─ manifest.json
├─ index.js
├─ template.html
├─ style.css
├─ README.md
└─ lib/
   ├─ storage-utils.js
   ├─ pinyin-utils.js
   ├─ vector-utils.js
   └─ zip-utils.js
```

根目录保留 SillyTavern 安装需要的入口文件。`lib/` 中是纯工具模块，不改变插件界面结构。

## 兼容说明

- 插件面板和注入逻辑依赖 SillyTavern 第三方扩展机制。
- API 接口按 OpenAI 兼容格式设计。
- 向量功能需要浏览器支持 IndexedDB。
- 若 SillyTavern 更新导致扩展接口变化，可能需要同步调整插件。
