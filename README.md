# Novel Injector

Novel Injector 是一个 SillyTavern 第三方扩展，用于把长篇小说整理成可注入的剧情、角色、阶段、世界设定、文风指南和向量索引。它的目标不是替 AI 写剧本，而是在发送消息时把当前需要的原著资料送入上下文，让续写更稳定地贴合原著时间线、人物关系、世界规则和当前剧情位置。

## 主要能力

- 上传 `.txt` 小说并按大小切分。
- 调用清洗 API 压缩正文，并提取剧情节点、角色人设、角色称呼、时间地点和支线关联。
- 将剧情划分为阶段，按阶段开启/关闭注入。
- 对指定阶段建立 Embedding 向量索引，并按最近对话语义召回相关原著片段。
- 对未向量化阶段直接注入剧情节点或压缩原文。
- 管理角色人设，支持启用/关闭角色注入、手动添加角色、AI 更新人设。
- 支持“用户代入角色”，把选中角色及勾选称呼在注入层替换为 `<user>`。
- 支持世界设定分类生成、编辑、启用/关闭。
- 支持文风指南生成、手动样本生成、指定分段生成与注入。
- 支持“穿书模式”，在剧情页管理当前节点、归档进度、推演后续行动方向。
- 支持外观配色主题，用户可通过颜色选择器调整主色、次色、强调、警示、背景和文字色，也可导入导出主题。
- 支持小说库快照、ZIP 导入导出、旧 JSON 导入、向量缓存清理和全量清理。

## 安装

### 通过 SillyTavern 扩展安装器

在 SillyTavern 的第三方扩展安装入口填入：

```text
https://github.com/ZZZa-o/novel-injector
```

安装后刷新 SillyTavern，点击顶部的书本图标打开插件面板。

### 手动安装

将本仓库文件夹放入 SillyTavern 的第三方扩展目录：

```text
SillyTavern/public/scripts/extensions/third-party/novel-injector
```

目录下应包含：

```text
manifest.json
index.js
template.html
style.css
lib/
README.md
```

刷新或重启 SillyTavern 后，在顶部扩展按钮中打开插件。

## 页面结构

插件底部导航包含五个页面：

| 页面 | 作用 |
|---|---|
| 处理 | 上传小说、清洗压缩、文风生成、向量化、注入参数配置 |
| 剧情 | 前文偏差分析、穿书模式、世界设定、关键剧情节点、阶段划分 |
| 角色 | 用户代入角色、角色人设、角色启用/关闭、AI 更新人设 |
| 阶段 | 阶段开关、阶段概括生成、未向量阶段注入模式、角色演绎提示词 |
| 设置 | 插件总开关、全局提示词、外观配色、小说库、导入导出、缓存清理 |

## 快速开始

1. 在“处理”页上传 `.txt` 小说。
2. 配置清洗 API：URL、Key、模型、超时、限速，必要时开启流式输出。
3. 点击“开始全自动清洗”。清洗会按分段依次生成压缩正文和结构化元数据。
4. 到“剧情”页检查时间轴、主线、支线和关键转折。
5. 在剧情页打开“划分阶段”，手动分配节点，或按转折点自动划分。
6. 在“处理”页的向量化面板选择阶段并开始向量化。
7. 在“阶段”页开启需要参与注入的阶段。
8. 在普通聊天中发送消息，插件会按配置注入剧情、角色、世界设定、文风和向量召回内容。
9. 需要穿书玩法时，在“剧情”页开启穿书模式，并在正文状态栏或弹窗中管理节点进度。

## API 配置

插件使用两类 OpenAI 兼容接口。

| 配置 | 用途 | 是否必需 |
|---|---|---|
| 清洗 API | 小说清洗压缩、剧情/角色提取、世界设定生成、文风生成、偏差分析、穿书推演 | 必需 |
| Embedding API | 阶段向量化、聊天时语义召回 | 可选 |

清洗 API 支持模型列表拉取、流式输出、超时设置和每分钟请求数限制。Embedding API 支持模型列表拉取和独立限速。

不使用向量化也可以使用插件。未向量化阶段会按“阶段”页设置，直接注入剧情节点或压缩原文。

## 清洗压缩

清洗会把每个小说分段转换为两部分：

1. 压缩正文：按时间顺序保留关键剧情、人物关系、时间地点、因果和转折。
2. `<ni_meta>`：结构化元数据。

当前清洗元数据包含：

```json
{
  "characters": [],
  "character_aliases": [],
  "plots": []
}
```

### characters

只记录本段首次登场角色。字段包括：

- `name`
- `role`
- `identity`
- `appearance`
- `gender`
- `personality`
- `relations`

角色人设以首次登场记录为准。后续分段不会覆盖已存在角色的人设字段，避免角色成长、改名或状态变化污染初次人设。

### character_aliases

用于收集角色称呼，不显示在角色人设页面。字段包括：

- `character_name`：称呼归属的角色。
- `text`：原文出现的称呼。
- `kind`：`primary`、`nickname`、`alias`、`stage_name`、`title`。
- `note`：简短说明。

可收集类型：正式名、常用昵称、外号、阶段性姓名、身份称谓。阶段性姓名例如长大后改名，`kind` 应为 `stage_name`。身份称谓如“少爷”“殿下”“师兄”会被记录为 `title`，但不会自动显示在角色人设正文里。

这些称呼会隐藏保存到角色对象的 `aliases` 字段，并在“用户代入角色”功能中作为可勾选候选。

### plots

剧情节点字段包括：

- `type`：`main`、`sub`、`pivot`。
- `title`
- `body`
- `sub_notes`
- `branch_links`
- `time`
- `location`
- `chunk_index`

`main` 和 `pivot` 可以通过 `branch_links` 关联 `sub` 节点。剧情页也提供 AI 修补关联按钮，以及支线编辑时手动选择关联剧情的能力。

## 剧情页

剧情页包含四块主要功能。

### 前文偏差分析

调用清洗 API，对比当前对话与当前激活阶段的原著参考内容，输出主线剧情、人物行为、地点信息、支线完整度等偏差分数和摘要。

### 穿书模式

穿书模式位于剧情页，并排在世界设定之前。它会围绕当前剧情节点管理推进状态。

可用能力：

- 开启/关闭穿书模式。
- 选择状态栏或弹窗显示剧情进度。
- 勾选节点完成状态，已完成节点会归档。
- 根据当前节点注入推进提示词或持续提示词。
- 推演后续行动方向，并可点击填入输入框。
- 暂停/继续节点推进。
- 编辑推进、持续、归纳、沉浸、后文推演提示词。

辅助开关：

| 开关 | 作用 |
|---|---|
| 归纳 | 在推进/持续提示词中要求 AI 生成对用户不可见的 `<ni_query>` 标签，并自动把 `ni_query` 加入召回消息标签 |
| 沉浸 | 追加信息边界，要求正文优先写 `<user>` 已感知、被告知或可合理推断的信息，避免提前揭示未公开姓名、身份、关系和秘密 |

### 世界设定

世界设定按分类管理。默认分类来自插件内置世界观分类，可逐类启用、关闭、编辑、重新生成，也可以添加新分类。

功能包括：

- AI 全部生成。
- 单类重新生成。
- 手动编辑分类内容。
- 分类开启/关闭注入。
- 添加自定义大类。

世界设定有独立注入位置、深度和消息角色配置。

### 关键剧情

剧情节点有四个视图：

- 时间轴
- 主线
- 支线
- 关键转折

可用操作：

- 展开/收起节点。
- 添加节点。
- 删除节点。
- 编辑节点。
- 修补支线关联。
- 支线编辑时选择关联的主线或转折剧情。
- 点击时间轴支线链接跳转到支线页。

剧情页还提供阶段划分面板，可以新建阶段、分配节点、查看未分配节点，并支持按转折点自动划分。

## 角色页

### 用户代入角色

“用户代入角色”位于角色人设上方。开启后，插件会在注入层把选中角色及已勾选称呼替换为 `<user>`。

特点：

- 角色列表来自清洗得出的角色。
- 称呼候选来自清洗出的 `character_aliases` 和角色本名。
- 称呼按出现阶段显示，例如“一阶段”“二阶段”。
- 用户可以手动添加、编辑、删除称呼。
- 每个聊天窗口独立保存称呼勾选状态。
- 支持“重置本聊天勾选”。
- 替换只发生在注入层，不改原文、不改清洗结果、不改向量数据。
- 不在检索层兼容原名和 `<user>`，避免 AI 把二者理解成两个人。
- 替换时会保护 `<...>` 标签内部内容，避免误改已有标签。

开启后，插件还会注入一条短说明，告诉 AI：`<user>` 代表选中原著角色，已勾选称呼都指向同一人，不要把原名或称呼写成另一个角色。

### 角色人设

角色页按角色类型分为：主角、配角、反派、其他。

可用操作：

- 开启/关闭单个角色注入。
- 查看/隐藏原始人设。
- 查看/隐藏 AI 更新人设。
- 手动编辑角色信息。
- 手动添加角色。
- 删除角色。
- 按当前分类全部开启/关闭。
- 按初次登场阶段批量开启/关闭角色。
- AI 更新已启用角色人设。

角色注入会使用已启用角色的身份、外貌、性格、关系等信息。隐藏的 `aliases` 不会作为角色人设正文显示。

## 阶段页

阶段页用于控制哪些剧情区间参与注入。

功能包括：

- 开启/关闭单个阶段。
- 全部开启/全部关闭。
- 显示每阶段向量化状态。
- 生成阶段标题和概括。
- 仅生成缺失概括的阶段。
- 编辑角色演绎提示词。
- 暂时关闭向量化注入。
- 选择未向量化阶段的注入模式。

未向量化阶段注入模式：

| 模式 | 说明 |
|---|---|
| 剧情节点 | 注入该阶段的主线、支线、转折节点，节省 token |
| 压缩原文 | 注入该阶段的清洗压缩正文，信息更完整但 token 消耗更高 |

切换到压缩原文模式时，插件会按需读取 `chunks.json`。

## 文风

文风面板位于处理页。

支持三种来源：

- 自动样本：从上传原文中截取样本。
- 手动范文：用户粘贴样本文本。
- 指定分段：选择某个小说分段作为样本。

文风提示词可编辑和恢复默认。生成结果可以收起/展开，并可通过注入设置开启或关闭文风注入。

## 向量化与召回

向量化会把已清洗的阶段文本切分为小块，调用 Embedding API 后写入浏览器 IndexedDB。

存储数据库：

```text
NovelInjectorVectors
```

每个向量块按小说 `novelKey` 隔离，不同小说不会混用。

召回配置包括：

- `Top K`
- 相似度阈值
- 召回时提取的消息标签
- 最近消息条数
- 向量注入位置
- 向量注入深度
- 向量消息角色

如果某阶段已向量化，聊天时会用最近对话构造查询并召回相关片段。如果阶段未向量化，则按阶段页设置直接注入剧情节点或压缩原文。

向量化面板还提供：

- 选择阶段。
- 全选/全不选/仅未向量。
- 向量诊断，查看 IndexedDB 中实际保存的向量数据。
- 补全向量。
- 清除当前小说向量缓存。

## 注入设置

处理页的“注入设置”可以分别配置以下内容的注入位置、深度和消息角色：

- 角色人设
- 阶段剧情
- 向量召回
- 世界设定
- 文风指南

可选注入位置包括主提示前、主提示后、聊天内等，具体效果取决于 SillyTavern 当前扩展提示词机制。插件总开关关闭时，会停止主要注入逻辑。

## 外观配色

设置页提供外观配色面板。面板默认收起，点击标题行可展开或折叠。

可配置的颜色包括：

- 主色。
- 次色。
- 强调。
- 警示。
- 背景。
- 文字。

主题下拉框支持内置主题和用户自定义主题。用户可以导入、导出、新建、删除和保存主题；颜色通过浏览器原生取色器选择。

“跟随美化”开启时，插件背景、文字颜色和毛玻璃效果跟随当前 SillyTavern 美化主题；关闭时，插件背景和文字使用外观配色中手动选择的颜色。

“去除边框”开启时，会隐藏插件大部分外层和内部控件边框，适合偏好更简洁界面的用户。小说库中的小说卡片边框会保留，便于区分快照。

## 小说库、导入导出与清理

设置页提供小说库和备份功能。

### 小说库

可将当前工作区保存为快照。快照包含剧情节点、角色人设、阶段、世界设定、文风、清洗状态、向量状态等索引信息。重数据会拆分写入 SillyTavern 的 `user/files`。

支持：

- 保存快照。
- 加载快照。
- 更新快照。
- 重命名快照。
- 删除快照及对应重数据。

### 导入导出

支持导出 ZIP，内容包括：

- `manifest.json`
- `settings.json`
- `chunks.jsonl`
- `vectors.bin`

导出不会包含 `cleanKey` 和 `vecKey`。

导入支持：

- 新版 ZIP。
- 旧版 JSON。

导入内容会作为新的小说库快照加入，不会直接覆盖当前工作区。需要手动加载导入的快照才会切换当前小说。

### 清理

支持：

- 清除当前小说向量缓存。
- 清除全部数据。

清除全部数据会删除当前小说的向量缓存、服务端重数据文件，并移除小说库中的当前条目。操作前会二次确认。

## 数据存储

| 数据 | 存储位置 |
|---|---|
| 轻量设置、API 配置、小说库索引、阶段开关、世界分类索引 | SillyTavern `extension_settings` |
| 剧情节点、角色人设、角色称呼、分段 meta、清洗状态、文风正文 | SillyTavern `user/files` 下的 `*_core.json` |
| 清洗压缩正文 | SillyTavern `user/files` 下的 `*_chunks.json` |
| 向量索引 | 浏览器 IndexedDB：`NovelInjectorVectors` |
| 穿书节点进度 | 当前聊天首条消息上的 `ni_tb` 数据 |
| 用户代入称呼勾选状态 | 当前聊天首条消息上的 `ni_user_sub.aliasStates` |

服务端重数据文件名使用拼音和随机 key，例如：

```text
ni_<快照名拼音>_<随机key>_core.json
ni_<快照名拼音>_<随机key>_chunks.json
```

这样做是为了兼容 SillyTavern 文件接口和跨设备迁移。

## 常见问题

### 不向量化能用吗？

可以。未向量化阶段会直接注入剧情节点或压缩原文。向量化主要用于长篇小说中更精确地召回相关片段。

### 修改阶段后需要重新向量化吗？

需要。阶段划分变化后，受影响阶段的向量状态会重置，需要重新选择这些阶段并向量化。

### 为什么用户代入角色没有自动出现很多称呼？

称呼来自清洗结果中的 `character_aliases`。旧清洗结果没有这个字段，需要重新清洗相关分段后才会出现。手动添加的称呼也可以立即使用。

### 用户代入会影响向量化和语义召回吗？

不会。替换只发生在注入层，不改清洗文本、剧情节点、角色人设和向量数据，也不在检索层把原名和 `<user>` 混合兼容。

### 角色称呼会显示在人设页面吗？

不会。称呼隐藏保存在角色对象的 `aliases` 字段，只用于用户代入角色功能。

### 支线为什么有“关联剧情”？

支线可以关联到某个主线或关键转折。时间轴会据此展示支线链接，也可以用 AI 修补关联自动补全 `branch_links`。

### 穿书模式在哪里？

穿书模式在“剧情”页，位于“世界设定”之前。正文里的状态栏或弹窗由穿书设置中的显示方式控制。

### 导入备份会覆盖当前小说吗？

不会。导入 ZIP 或旧 JSON 会创建一个新的小说库快照。只有手动加载该快照后，当前工作区才会切换。

### 导出会包含 API Key 吗？

不会。导出逻辑会跳过 `cleanKey` 和 `vecKey`。

### 清洗失败怎么办？

检查清洗 API 的 URL、Key、模型、超时和限速。单段失败后可以单独生成该段，也可以使用“重试失败分段”。

### 向量化失败怎么办？

检查 Embedding API 的 URL、Key 和模型名。遇到限速时降低每分钟请求数后重试。也可以用向量诊断确认 IndexedDB 中实际保存的数据。

## 兼容说明

- 插件依赖 SillyTavern 第三方扩展机制。
- 清洗 API 使用 OpenAI 兼容 Chat Completions 风格接口。
- Embedding API 使用 OpenAI 兼容 Embeddings 风格接口。
- 向量数据依赖浏览器 IndexedDB。
- 若 SillyTavern 的扩展提示词接口发生变化，注入位置和深度行为可能需要同步调整。
